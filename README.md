# Console-file-manager

 Консольный файловый менеджер начального уровня, который покрывает минимальный набор функционала по работе с файлами.
 (см. [ТЗ](https://docs.google.com/document/d/13Vps0Eqm4_6Q9xJHWAcSY2ZQ5Hc12HQdFMWa4WsRua0/edit),

 [Описание-2](https://docs.google.com/presentation/d/1k-l0I93pMlS1FNHTFnMzcwGgJBNtpGeV5q7Mw7_Ie50/edit#slide=id.g819c2089a5_2_41)
 )

## Функции
1. Просмотр файловой структуры
2. Копирование файлов, каталогов.
3. Удаление файлов, каталогов.
4. Получение информации о размерах, системных атрибутов файла, каталога.

## Описание
Окно приложения состоит из трех "фреймов" (сверху вниз):
1. Фрейм для отображения файловой структуры.
2. Фрейм для отображения текущих cообщений приложения и информации о файлах.
3. Фрейм командной строки.

Фрейм файловой структуры отображает структуру постранично, с возможностью отображения конкретной страницы (см. команду "ls").
Фрейм для отображения информации показывает актуальную информацию - например информацию о файлах, которые указал пользователь (см. команды "dir", "file")
или сообщения приложения (например об ошибках)
Фрейм командной строки содержит командную строку для ввода консольных команд (см. "Команды").

## Использование

Для выполнения действия необходимо ввести в командную строку соответсвующую команду с аргументами.

### Команды 
1. **help** - Показать список команд
2. **ex**   - Выйти из приложения
3. **ls**   - Отобразить файловую структуру каталога, аргументы - путь к каталогу, и номер страницы (-p(NumberPage))  (Пример: _ls Disk:\source  -p(NumberPage)_)
4. **file** - Отобразить информацию о файле, аргументы - путь к файлу (Пример: _ls Disk:\source\file.ext_)
5. **dir**  - Отобразить информацию о каталоге, аргументы - путь к каталогу (Пример: _ls Disk:\source_)
6. **cp**  - Копировать файл(каталог) из ... в ..., аргументы - путь к источнику, путь к назначению (Пример: _cp Disk:\source  Disk:\dest_ )
7. **rm**  - Удалить файл(каталог) , аргументы - путь к источнику (Пример: _rm Disk:\source_ )

## Известные проблемы

Проблема не проявляется, если запускать приложение в режиме дебага в VisualStudio.
Мы работаем над устранением этой проблемы.
